High Efficiency Web Application Framework/Server: Beego

Beego is an open source framework to build and develop web applications powered by golang, you can find more info about it on its homepage. Its repo on GitHub has got 3000+ stars yet.

As I mentioned in title above, it’s a server too just like node and tornado(actually it’s inspired by tornado, which meets my flavor too). I’ll tell you more about it later.

Why Beego

I’ve been working on web/app server for a year around and have used many framework like spring, web.py, tornado and node. Well python is cool and most of my commercial project was based on web.py and tornado then. I’ve been using Beego for 7 month and just can’t love it more. How does it hold me so tight?

Golang

Last year Nov, a friend working for Google told me that they’ve already rebased many internal projects using Golang and they works with high efficiency, contributed by the concurrence supported from Golang in language level.

Tis raised my interest to find out what’s Golang and how it works. After taking a glance at this language, I’ve wrote an article about it, I’ll post it there later.

REST-ful

I always work on projects in small team or sometimes do it on my own, so agile development means a lot to me. How to make the C/S or B/S development separated to meet the needs of agile dev?

I don’t wanna mess with any works that other developers in team are working on and we all want a clean and nice code style. REST-ful structure just allow you to do own things and ignore any implementation details that other side do. All we need is the API Doc to make the process run well.

Beego’s design is based on the REST-ful web framework and you can work with it well and sharp. The features are just full of the soul of API designs such as the namespace router and the API application create.

When Golang came out, we all know dat it aims at to be the next C. Well it may be designed to fit its ambitions but, in da other hands, is Golang really fit to be a web development language?

Come back to Beego, which make the Golang with the ability of server stuff handling just amazing.

Easy To Go With Bee

Bee tool is a project for help develop beego rapidly. With bee tool you can create, auto compile and reload, develop, test, and deploy beego applications easily. (sorry for just copied lol

I’m about to pick up one of my favorite command — bee api.

It allows you to build a api web app based on your sql database. How simple it is?Assume you have a database called beeDB with several datas and you wanna build a web app called beeAPP. All you need to do is run this command:

# bee api beeAPP -conn=root:123@tcp\(127.0.0.1:3306)/beeDB

Ok, it’s done. Now you have a complete web app with test framework and auto generated API docs(using swagger).

But how it works?Actually it generates into your database and construct the controllers and models via the pre-designed template. There’s a sample video by the Beego’s author shows you how to do this. It’s kinda cool isn’t it?

REST-ful Router Style

With the implicit supports of RESTful, Beego supplies many router method like fixed router, regex router and even you can customize your own rules like:

beego.Router("/api/list",&RestController{},"*:ListFood")
beego.Router("/api/create",&RestController{},"post:CreateFood")
beego.Router("/api/update",&RestController{},"put:UpdateFood")
beego.Router("/api/delete",&RestController{},"delete:DeleteFood")

As you can see da third para is what you want to call your API in controller. It’s just convenient.

Also it supports annotation routers and namespace, which I love most. Check the router code:

//init namespace
ns := 
beego.NewNamespace("/v1",
    beego.NSCond(func(ctx *context.Context) bool {
        if ctx.Input.Domain() == "api.beego.me" {
            return true
        }
        return false
    }),
    beego.NSBefore(auth),
    beego.NSGet("/notallowed", func(ctx *context.Context) {
        ctx.Output.Body([]byte("notAllowed"))
    }),
    beego.NSRouter("/version", &AdminController{}, "get:ShowAPIVersion"),
    beego.NSRouter("/changepassword", &UserController{}),
    beego.NSNamespace("/shop",
        beego.NSBefore(sentry),
        beego.NSGet("/:id", func(ctx *context.Context) {
            ctx.Output.Body([]byte("notAllowed"))
        }),
    ),
    beego.NSNamespace("/cms",
        beego.NSInclude(
            &controllers.MainController{},
            &controllers.CMSController{},
            &controllers.BlockController{},
        ),
    ),
)

//register namespace
beego.AddNamespace(ns)

The structrue looks good and you can set your router clean and clear with filter/auth/context and more.

Stand Alone Deployment

Golang is a compiled language, so you can just put the binary file of your Beego app on server and run it as a daemon.

We all know the interpreter language powered application just can not get the code protected well, and also there’s a lot other problem with security.

You can deploy your app more flexibly with supervisord, nginx or apache, and you sleep well.
